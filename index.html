Why do I have to open claude.ai from this page?
Browsers isolate tabs opened directly by the user (typing a URL, clicking a bookmark) into separate browsing-context groups for security. Cross-origin postMessage needs an explicit link between two tabs. When this page calls window.open('https://claude.ai'), the child inherits a window.opener reference back to this tab — which the snippet uses to push updates without stealing focus.
What does the snippet do? Is it safe?
The snippet reads your lastActiveOrg cookie (an organization ID, not a secret) and calls claude.ai/api/organizations/<org>/usage using your existing logged-in session — the same request claude.ai makes to render its sidebar widget. The response is sent to this page via postMessage (or URL hash on first contact). No data is sent to any third-party server. Your Claude session cookie never leaves claude.ai.
How do I stop the auto-refresh?
Either close the claude.ai tab, or run clearInterval(__cuw.iv) in the same console where you started it. Reloading the claude.ai tab also stops it — you'd re-open via the button and re-paste to resume.
